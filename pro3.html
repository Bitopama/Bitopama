E Commerce Traffic Analysis

E-Commerce Website Traffic Analysis

The primary goal of this project was to analyze website traffic data for an e-commerce platform to derive actionable insights related to user behavior, conversion rates, and overall site performance. The objective was to understand traffic sources, user engagement, and the impact of product launches on sales.

Channel Portfolio and Conversion Insights

Introduction

This analysis focused on evaluating the performance of various marketing channels, emphasizing user engagement via session trends and conversion rates for an e-commerce platform. The analysis included traffic sources, device types, and campaign strategies, using MySQL for data extraction and manipulation.

1. Channel Portfolio Analysis

To assess marketing channels' effectiveness, I analyzed UTM content to identify channels with the highest session counts and orders.

SELECT ws.utm_content, COUNT(DISTINCT ws.website_session_id) AS sessions, 
       COUNT(DISTINCT o.website_session_id) AS orders,
       (COUNT(DISTINCT o.website_session_id) / COUNT(DISTINCT ws.website_session_id)) AS conversion_rate
FROM website_sessions ws
LEFT JOIN orders o ON ws.website_session_id = o.website_session_id
WHERE ws.created_at BETWEEN '2014-01-01' AND '2014-02-01'
GROUP BY ws.utm_content
ORDER BY conversion_rate DESC;

Findings: Certain UTM content types were top performers in generating sessions and orders, particularly Google Ad-1. The conversion rates highlighted the value of targeted messaging.

2. Session Volume Trends: Google Search vs. Bing Search

To compare Google Search and Bing Search session volume trends weekly for non-brand campaigns:

SELECT MIN(DATE(created_at)) AS start_of_week,
       COUNT(DISTINCT CASE WHEN utm_source = "gsearch" THEN website_session_id END) AS gsearch,
       COUNT(DISTINCT CASE WHEN utm_source = "bsearch" THEN website_session_id END) AS bsearch
FROM website_sessions 
WHERE created_at BETWEEN '2012-08-22' AND '2012-11-29' 
      AND utm_campaign = "nonbrand"
GROUP BY WEEK(created_at);

Findings: Google Search consistently outperformed Bing in session volume, showing its value as a primary traffic source for non-brand campaigns.

3. Device Type Analysis for Non-Brand Campaigns

Session volume segmented by device type for Google and Bing searches:

SELECT utm_source,
       COUNT(DISTINCT CASE WHEN device_type = "mobile" THEN website_session_id END) AS mobile_sessions,
       COUNT(DISTINCT CASE WHEN device_type = "desktop" THEN website_session_id END) AS desktop_sessions
FROM website_sessions
WHERE created_at BETWEEN '2012-08-22' AND '2012-11-30' 
      AND utm_campaign = "nonbrand"
GROUP BY utm_source;

Findings: Higher sessions on desktop for Google Search.

4. Conversion Rates by Device Type

Exploring conversion rates by device type for Google and Bing searches:

SELECT ws.device_type, ws.utm_source,
       COUNT(DISTINCT ws.website_session_id) AS sessions,
       COUNT(DISTINCT o.order_id) AS orders,
       COUNT(DISTINCT o.website_session_id) / COUNT(DISTINCT ws.website_session_id) AS conversion_rates
FROM website_sessions ws
LEFT JOIN orders o ON ws.website_session_id = o.website_session_id
WHERE ws.created_at >= '2012-08-22' AND ws.created_at < '2012-09-18'
      AND ws.utm_campaign = "nonbrand"
GROUP BY ws.device_type, ws.utm_source;

Findings: Overall desktop sessions surpassed mobile sessions with Google Search having the highest conversion rate.

5. Direct Traffic and Conversion Analysis

Analyzing direct traffic conversion rates by month:

SELECT YEAR(created_at) AS yr, MONTH(created_at) AS mo,
       COUNT(DISTINCT CASE WHEN utm_campaign = "nonbrand" THEN website_session_id END) AS nonbrand,
       COUNT(DISTINCT CASE WHEN utm_campaign = "brand" THEN website_session_id END) AS brand,
       COUNT(DISTINCT CASE WHEN utm_campaign = "brand" THEN website_session_id END) / 
       COUNT(DISTINCT CASE WHEN utm_campaign = "nonbrand" THEN website_session_id END) AS brand_pct_of_nonbrand,
       COUNT(DISTINCT CASE WHEN utm_source IS NULL AND http_referer IS NULL THEN website_session_id END) AS direct_type,
       COUNT(DISTINCT CASE WHEN utm_source IS NULL AND http_referer IS NULL THEN website_session_id END) / 
       COUNT(DISTINCT CASE WHEN utm_campaign = "nonbrand" THEN website_session_id END) AS direct_pct_of_nonbrand,
       COUNT(DISTINCT CASE WHEN utm_source IS NULL AND http_referer = "https://www.gsearch.com" THEN website_session_id END) AS organic_g_type,
       COUNT(DISTINCT CASE WHEN utm_source IS NULL AND http_referer = "https://www.gsearch.com" THEN website_session_id END) / 
       COUNT(DISTINCT CASE WHEN utm_campaign = "nonbrand" THEN website_session_id END) AS direct_og_of_nonbrand,
       COUNT(DISTINCT CASE WHEN utm_source IS NULL AND http_referer = "https://www.bsearch.com" THEN website_session_id END) AS organic_b_type,
       COUNT(DISTINCT CASE WHEN utm_source IS NULL AND http_referer = "https://www.bsearch.com" THEN website_session_id END) / 
       COUNT(DISTINCT CASE WHEN utm_campaign = "nonbrand" THEN website_session_id END) AS direct_ob_of_nonbrand
FROM website_sessions
GROUP BY 1, 2;

Findings: Direct traffic as a fraction of non-brand campaigns grew from 5% in 2012 to 16% in 2015, indicating stronger user engagement and loyalty. Similarly, an increase in organic search traffic over the years reflects improved brand from 0.38% to 15% recognition and visibility.

Conclusion

Through this analysis of traffic sources, device types, and direct interactions, I identified key areas for optimization and strategy enhancement. Focusing on Google Search strategies, mobile optimization, and direct traffic engagement could drive significant growth in conversions.

Bounce Rate Trends and Funnel Analysis

Introduction

In this segment of the analysis, I focused on understanding user engagement through bounce rates from landing pages and examining the conversion funnel for the product "Original Mr. Fuzzy." This investigation aimed to identify user behavior patterns and drop-off points during the shopping experience.

1. Bounce Rate Weekly Trends from Landing Pages

To evaluate user engagement, I analyzed bounce rates across various landing pages driven by Google Search for non-brand campaigns. 

WITH req_sess AS (
    SELECT ws.website_session_id, ws.created_at, wp.pageview_url, wp.website_pageview_id
    FROM website_sessions ws 
    LEFT JOIN website_pageviews wp ON ws.website_session_id = wp.website_session_id 
    WHERE ws.created_at < '2012-08-31' AND ws.created_at > '2012-06-01' 
          AND ws.utm_source = "gsearch" AND ws.utm_campaign = "nonbrand"
),
land_page AS (
    SELECT MIN(website_pageview_id) AS min_page, website_session_id 
    FROM req_sess
    GROUP BY website_session_id
),
land_url AS (
    SELECT r.website_session_id, r.pageview_url 
    FROM land_page lp 
    LEFT JOIN req_sess r ON lp.min_page = r.website_pageview_id
),
bounce AS (
    SELECT r.website_session_id, COUNT(DISTINCT r.website_pageview_id) AS page_num, l.pageview_url AS num_page 
    FROM req_sess r
    LEFT JOIN land_url l ON r.website_session_id = l.website_session_id
    GROUP BY r.website_session_id, l.pageview_url
    HAVING COUNT(DISTINCT r.website_pageview_id) = 1
)
SELECT l.pageview_url, MIN(DATE(r.created_at)) AS first_date, 
       COUNT(DISTINCT l.website_session_id) AS total,
       COUNT(DISTINCT b.website_session_id) AS bou,
       COUNT(DISTINCT b.website_session_id) / COUNT(DISTINCT l.website_session_id) AS bou_rate
FROM req_sess r
LEFT JOIN land_url l ON r.website_session_id = l.website_session_id
LEFT JOIN bounce b ON r.website_session_id = b.website_session_id
GROUP BY l.pageview_url, WEEK(r.created_at);

Findings: The analysis indicated varying bounce rates across different landing pages.

2. Funnel Analysis of "Original Mr. Fuzzy"

To identify potential drop-off points in the user journey for the "Original Mr. Fuzzy" product, I performed a funnel analysis to track user interactions from product selection to order completion. 

            WITH main_table AS (
                SELECT ws.website_session_id, wp.website_pageview_id, wp.created_at AS pv_created_at, wp.pageview_url,
                       CASE WHEN wp.pageview_url IN ('/home', '/lander-1', '/lander-2') THEN 1 ELSE 0 END AS start_click,
                       CASE WHEN wp.pageview_url = '/products' THEN 1 ELSE 0 END AS product_click,
                       CASE WHEN wp.pageview_url = '/the-original-mr-fuzzy' THEN 1 ELSE 0 END AS mr_fuzzy_click,
                       CASE WHEN wp.pageview_url = '/cart' THEN 1 ELSE 0 END AS cart_click,
                       CASE WHEN wp.pageview_url = '/shipping' THEN 1 ELSE 0 END AS ship_click,
                       CASE WHEN wp.pageview_url IN ('/billing') THEN 1 ELSE 0 END AS bill_click,
                       CASE WHEN wp.pageview_url = '/thank-you-for-your-order' THEN 1 ELSE 0 END AS ty_click
                FROM website_sessions ws
                LEFT JOIN website_pageviews wp ON ws.website_session_id = wp.website_session_id
                WHERE ws.created_at BETWEEN '2012-08-05' AND '2012-09-05'
                      AND wp.pageview_url IN ('/home', '/lander-1', '/lander-2', '/products', '/the-original-mr-fuzzy', '/cart', '/shipping', '/billing', '/thank-you-for-your-order')
                      AND ws.utm_source = "gsearch"
            ),
            funnel AS (
                SELECT website_session_id, 
                       MAX(product_click) AS pc, 
                       MAX(mr_fuzzy_click) AS fc, 
                       MAX(cart_click) AS cc,
                       MAX(ship_click) AS shc, 
                       MAX(bill_click) AS bc, 
                       MAX(ty_click) AS tyc
                FROM main_table
                GROUP BY website_session_id
            )
            SELECT COUNT(DISTINCT website_session_id) AS total_sessions,
                   SUM(pc) AS conv_to_pc,
                   SUM(fc) AS conv_to_fc,
                   SUM(cc) AS conv_to_cc,
                   SUM(shc) AS conv_to_shc,
                   SUM(bc) AS conv_to_bc,
                   SUM(tyc) AS conv_to_tyc,
                   SUM(fc) / SUM(pc) AS rate_fuzzy,
                   SUM(cc) / SUM(fc) AS rate_cart,
                   SUM(shc) / SUM(cc) AS rate_ship,
                   SUM(bc) / SUM(shc) AS rate_bill,
                   SUM(tyc) / SUM(bc) AS rate_order
            FROM funnel;

Findings: The funnel analysis revealed critical insights into user drop-off points.The lowest conversion rate was observed between the billing page and the order confirmation page(about 60% drop-off), indicating potential friction or issues on the billing page. To address this, two alternative billing page designs were introduced to simplify the process and improve user flow.

E-Commerce Website Traffic Analysis: Impact of Product Launch on Conversion Rates

Introduction

This section focuses on the impact of the product launch of "Love Bear" on user sessions and conversion rates for the products "Mr. Fuzzy" and "Love Bear." By analyzing user behavior before and after the product launch, I aimed to understand how the introduction of a new product influences customer engagement and purchasing decisions.

1. Impact of Product Launch on Session Navigation

To evaluate how the launch of "Love Bear" affected user navigation from the product page to subsequent pages, I analyzed user sessions over the defined period. 

WITH product_sessions AS (
    SELECT COUNT(DISTINCT website_pageview_id) AS page_count, website_session_id
    FROM website_pageviews
    WHERE created_at > '2012-10-06' AND created_at < '2013-04-06'
    GROUP BY website_session_id
    HAVING COUNT(DISTINCT website_pageview_id) > 2
),
non_bounced_sessions AS (
    SELECT COUNT(DISTINCT website_pageview_id) AS page_count, website_session_id
    FROM website_pageviews
    WHERE created_at > '2012-10-06' AND created_at < '2013-04-06'
    GROUP BY website_session_id
    HAVING COUNT(DISTINCT website_pageview_id) > 1
),
products AS (
    SELECT website_session_id, pageview_url
    FROM website_pageviews
    WHERE pageview_url IN ('/the-original-mr-fuzzy', '/the-forever-love-bear')
)

SELECT 
    CASE 
        WHEN ws.created_at < '2013-01-06' THEN 'pre_product_2' 
        WHEN ws.created_at >= '2013-01-06' THEN 'post_product_2' 
    END AS time_period,
    COUNT(DISTINCT nbs.website_session_id) AS sessions,
    COUNT(DISTINCT ps.website_session_id) AS sessions_w_next_pg,
    COUNT(DISTINCT ps.website_session_id) / COUNT(DISTINCT nbs.website_session_id) AS pct_w_next_pg,
    COUNT(DISTINCT CASE WHEN p.pageview_url = '/the-original-mr-fuzzy' THEN p.website_session_id END) AS to_mrfuzzy,
    COUNT(DISTINCT CASE WHEN p.pageview_url = '/the-forever-love-bear' THEN p.website_session_id END) AS to_lovebear
FROM website_sessions ws
LEFT JOIN non_bounced_sessions nbs ON nbs.website_session_id = ws.website_session_id
LEFT JOIN product_sessions ps ON ps.website_session_id = ws.website_session_id
LEFT JOIN products p ON p.website_session_id = ws.website_session_id
WHERE ws.created_at > '2012-10-06' AND ws.created_at < '2013-04-06'
GROUP BY 1;

Findings: The analysis revealed significant shifts in user behavior pre- and post-launch of "Love Bear". Three months prior to the product launch, 73% of users clicked from the product page to the product options, and this figure increased to 76.5% three months after the launch. This indicates a positive response to the new product, encouraging users to explore further.

2. Funnel Analysis of "Mr. Fuzzy" and "Love Bear"

To assess the impact of the "Love Bear" launch on conversion rates, I performed a funnel analysis comparing user interactions before and after the launch. 

WITH product_sessions AS (
    SELECT COUNT(DISTINCT website_pageview_id) AS page_count, website_session_id
    FROM website_pageviews
    WHERE created_at > '2012-10-06' AND created_at < '2013-04-06'
    GROUP BY website_session_id
    HAVING COUNT(DISTINCT website_pageview_id) > 2
),
funnel_1 AS (
    SELECT website_session_id, created_at,
           CASE WHEN pageview_url = '/cart' THEN 1 ELSE 0 END AS to_cart,
           CASE WHEN pageview_url = '/shipping' THEN 1 ELSE 0 END AS to_ship,
           CASE WHEN pageview_url IN ('/billing', '/billing-1', '/billing-2') THEN 1 ELSE 0 END AS to_bill,
           CASE WHEN pageview_url = '/thank-you-for-your-order' THEN 1 ELSE 0 END AS to_thanks
    FROM website_pageviews
    WHERE website_session_id IN (SELECT website_session_id FROM product_sessions)
)

SELECT
    CASE 
        WHEN wp.pageview_url = '/the-original-mr-fuzzy' THEN 'mrfuzzy' 
        WHEN wp.pageview_url = '/the-forever-love-bear' THEN 'lovebear' 
    END AS product_category,
    COUNT(DISTINCT f1.website_session_id) AS viewed_the_product,
    COUNT(DISTINCT CASE WHEN f1.to_cart = 1 THEN wp.website_session_id END) AS to_cart,
    COUNT(DISTINCT CASE WHEN f1.to_ship = 1 THEN wp.website_session_id END) AS to_ship,
    COUNT(DISTINCT CASE WHEN f1.to_bill = 1 THEN wp.website_session_id END) AS to_bill,
    COUNT(DISTINCT CASE WHEN f1.to_thanks = 1 THEN wp.website_session_id END) AS to_thanks,
    COUNT(DISTINCT CASE WHEN f1.to_thanks = 1 THEN wp.website_session_id END) / COUNT(DISTINCT f1.website_session_id) AS buying_to_viewing
FROM website_pageviews wp
JOIN funnel_1 f1 ON wp.website_session_id = f1.website_session_id 
WHERE wp.created_at > '2012-10-06' AND wp.created_at < '2013-04-06'
AND pageview_url IN ('/the-original-mr-fuzzy', '/the-forever-love-bear')
GROUP BY 1;

Findings: The funnel analysis illustrated how the introduction of "Love Bear" influenced conversion rates.fter conducting the funnel analysis from product viewing to purchase, it was found that while the older product, "The Original Mr. Fuzzy," had a greater total number of orders, the newly launched product, "The Forever Love Bear," achieved a higher purchase-to-viewing conversion rate of 18%, compared to 14% for the older product.This data is essential in understanding the effectiveness of the new product in driving sales and customer engagement.

3. Revenue and Margin Analysis

Finally, I conducted a revenue and margin analysis to assess financial performance before and after the launch of "Love Bear." 

SELECT
    CASE 
        WHEN ws.created_at < '2013-01-06' THEN 'pre_love_bear'
        WHEN ws.created_at >= '2013-01-06' THEN 'post_love_bear' 
    END AS time_period,
    COUNT(DISTINCT o.order_id) / COUNT(DISTINCT ws.website_session_id) AS conv_rate,
    SUM(o.price_usd) / COUNT(DISTINCT o.order_id) AS aov,
    SUM(o.price_usd - o.cogs_usd) / COUNT(DISTINCT o.order_id) AS avg_order_margin,
    SUM(o.price_usd) / COUNT(DISTINCT ws.website_session_id) AS rev_per_sess,
    SUM(o.price_usd - o.cogs_usd) / COUNT(DISTINCT ws.website_session_id) AS margin_per_sess 
FROM website_sessions ws
LEFT JOIN orders o ON ws.website_session_id = o.website_session_id
WHERE ws.created_at BETWEEN '2012-10-06' AND '2013-04-06'
GROUP BY 1;

Findings: The analysis revealed notable differences in conversion rates and average order values before and after the launch of "Love Bear". Following the product launch, the average margin per order rose from $30.50 to $32.00, while the average revenue per order increased from $50.00 to $52.20. These changes indicate a positive impact resulting from the product launch. This highlights the positive impact of the product launch on the financial metrics of the e-commerce site.

Conclusion

Through the analysis of user sessions, funnel conversion rates, and revenue margins, I was able to demonstrate the significant impact of the "Love Bear" product launch on customer behavior and business performance. These insights can guide future product launches and marketing strategies to optimize user engagement and sales.

Repeat Sessions and Refund Rates Analysis

Introduction

This analysis focuses on user behavior regarding repeat sessions, including the timing between these sessions and insights into new and repeated sessions based on traffic sources. Additionally, I examined refund rates for specific products over time to understand the potential correlation between repeat sessions and customer satisfaction.

1. Repeat Sessions Analysis

To understand how many users engage in repeat sessions, I created a temporary table to capture user IDs and their corresponding sessions. 

CREATE TEMPORARY TABLE sessions_w_repeats AS
SELECT ns.user_id,
       ns.website_session_id AS new_session_id,
       ns.created_at AS new_session_created_at,
       ws.created_at AS repeated_session_created_at,
       ws.website_session_id AS repeat_sessions
FROM (
    SELECT user_id,
           website_session_id, created_at 
    FROM website_sessions
    WHERE created_at < '2014-11-01'
      AND created_at >= '2014-01-01'
      AND is_repeat_session = 0
) AS ns
LEFT JOIN website_sessions ws
ON ws.website_session_id > ns.website_session_id
   AND ws.user_id = ns.user_id
   AND ws.is_repeat_session = 1
   AND ws.created_at < '2014-11-01' 
   AND ws.created_at >= '2014-01-01';

-- Retrieve data for review
SELECT * FROM sessions_w_repeats;

-- Count distinct users for each repeat session
SELECT repeat_sessions,
       COUNT(DISTINCT user_id) AS users
FROM (
    SELECT user_id,
           COUNT(DISTINCT repeat_sessions) AS repeat_sessions
    FROM sessions_w_repeats
    GROUP BY 1
) AS user_level
GROUP BY 1;

-- Example: Review a specific user
SELECT * FROM sessions_w_repeats WHERE user_id = 152837;

Findings: This analysis allowed me to quantify how many users engage in repeat sessions.The data shows that there were 126,813 users with zero repeat sessions, while only 14,086 users had one repeat session. Notably, just 315 users had two repeat sessions, and 4,686 users had three repeat sessions..Understanding user engagement is crucial for optimizing retention strategies.

2. Time Between First and Second Sessions

Next, I investigated the time interval between a user's first and second session. 

WITH first_t AS (
    SELECT user_id, new_session_created_at, repeated_session_created_at 
    FROM sessions_w_repeats
    WHERE repeat_sessions IS NOT NULL
),
data_t AS (
    SELECT user_id, 
           DATEDIFF(MIN(repeated_session_created_at), MIN(new_session_created_at)) AS diff_time
    FROM first_t
    GROUP BY 1
)
SELECT AVG(diff_time) AS avg_time_diff,
       MIN(diff_time) AS min_time_diff,
       MAX(diff_time) AS max_time_diff 
FROM data_t;

Findings: The analysis revealed insights into user behavior patterns concerning repeat sessions. The average time between sessions was 33.25 days, with a minimum interval of 1 day and a maximum of 69 days. This range indicates variability in user engagement, suggesting potential differences in how frequently users return.

3. New vs. Repeated Sessions Based on Traffic Sources

To analyze how different traffic sources contributed to new and repeat sessions, I classified sessions based on UTM parameters. 

SELECT 
    CASE 
        WHEN utm_source IS NULL AND http_referer IN ('https://www.gsearch.com','https://www.bsearch.com') THEN 'organic search' 
        WHEN utm_source IS NULL AND http_referer IS NULL THEN 'direct type in'
        WHEN utm_campaign = 'nonbrand' THEN 'paid nonbrand'
        WHEN utm_campaign = 'brand' THEN 'paid brand'

    END AS traffic_source,
    COUNT(DISTINCT CASE WHEN is_repeat_session = 0 THEN website_session_id END) AS new_sessions,
    COUNT(DISTINCT CASE WHEN is_repeat_session = 1 THEN website_session_id END) AS repeat_sessions
FROM website_sessions
WHERE created_at BETWEEN '2014-01-01' AND '2014-11-01'
GROUP BY 1;

Findings: This analysis provided insights into traffic source effectiveness in attracting new versus repeat sessions. Organic searches, followed by brand campaign visits and then direct URL entries, had the highest number of repeated sessions. This pattern indicates strong brand penetration, increased website engagement, and effective user retention, all of which are beneficial for the company.

Cross-Selling and Revenue Impact

Introduction

This analysis examines the effectiveness of cross-selling strategies for various product types, exploring user behavior in relation to cart sessions, and assessing the financial impact of the "Birthday Sugar Panda" product launch. Key focus areas include conversion rates and revenue changes.

1. Cross-Selling Product Analysis

To evaluate cross-selling opportunities, I analyzed order data to determine how frequently distinct primary products were purchased along with cross-sold items. 

SELECT o.primary_product_id,
       COUNT(DISTINCT oi.order_item_id) AS order_counts,
       COUNT(DISTINCT CASE WHEN oi.product_id = 1 THEN oi.order_item_id END) AS cross_sell_prod_1,
       COUNT(DISTINCT CASE WHEN oi.product_id = 2 THEN oi.order_item_id END) AS cross_sell_prod_2,
       COUNT(DISTINCT CASE WHEN oi.product_id = 3 THEN oi.order_item_id END) AS cross_sell_prod_3
FROM orders o
LEFT JOIN order_items oi ON o.order_id = oi.order_id 
WHERE o.order_id BETWEEN 10000 AND 11000
GROUP BY 1;

Findings: This analysis highlighted the most common cross-sold products alongside each primary product. Overall, Product 1 had the highest sales and was frequently cross-sold with Products 2 and 3, with Product 1 as the primary item. In contrast, when Products 2 or 3 were the primary items, the other two products were not often purchased together.
